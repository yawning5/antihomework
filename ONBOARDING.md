# 온보딩 문서 초안

## 1. 프로젝트 개요

이 프로젝트는 Windows 환경에서 PC 카카오톡 채팅방을 찾아 메시지를 전송하는 콘솔 프로그램입니다.
카카오톡 공식 API를 사용하는 방식이 아니라, Win32 API를 이용해 열려 있는 창과 입력 컨트롤을 탐색하고 제어합니다.

현재 기준 핵심 기능은 두 가지입니다.

- 열려 있는 카카오톡 채팅방 목록 조회
- 선택한 채팅방으로 메시지 전송

## 2. 실행 환경

- 운영체제: Windows 10/11
- SDK: .NET 8
- 대상 앱: PC 카카오톡 로그인 상태
- 전제 조건: 메시지를 보낼 채팅방이 팝업 창으로 열려 있어야 함

## 3. 프로젝트 구조

```text
src/KakaoTalkAutomation/
├── Program.cs
├── ChatFinder.cs
├── MessageSender.cs
├── Win32.cs
└── KakaoTalkAutomation.csproj
```

각 파일의 역할은 아래와 같습니다.

- `Program.cs`
  - 콘솔 메뉴 진입점
  - 채팅방 목록 조회와 메시지 전송 흐름을 연결
- `ChatFinder.cs`
  - 카카오톡 프로세스를 찾고, 열린 채팅방 창을 수집
- `MessageSender.cs`
  - 채팅방 입력 컨트롤을 찾고 메시지 전송 수행
- `Win32.cs`
  - `user32.dll` 기반 Win32 API 선언과 보조 메서드 제공

## 4. 실행 방법

저장소 루트 기준:

```bash
dotnet build src/KakaoTalkAutomation/KakaoTalkAutomation.csproj
dotnet run --project src/KakaoTalkAutomation
```

실행 시 표시되는 메뉴:

```text
=== 카카오톡 자동화 프로그램 ===

1. 채팅방 목록 보기
2. 메시지 보내기
0. 종료
```

## 5. 동작 방식 요약

### 채팅방 탐색

`ChatFinder.Find()`는 아래 순서로 동작합니다.

1. `Process.GetProcessesByName("KakaoTalk")`로 카카오톡 프로세스 ID를 수집
2. `EnumWindows`로 모든 최상위 창을 순회
3. 카카오톡 프로세스에 속하고, 현재 보이는 창만 필터링
4. 제목이 비어 있지 않고 `"카카오톡"`이 아닌 창을 채팅방으로 간주

### 메시지 전송

`MessageSender.Send()`는 아래 순서로 동작합니다.

1. 대상 채팅방 창을 앞으로 가져옴
2. `EnumChildWindows`로 입력 가능한 Edit 계열 컨트롤 탐색
3. 메시지를 클립보드에 복사
4. `Ctrl+V`와 `Enter` 키 입력을 시뮬레이션해 전송

중요한 점은 직접 텍스트를 세팅하는 방식이 아니라, 클립보드 붙여넣기 방식에 의존한다는 것입니다.

## 6. 신규 참여자가 먼저 이해해야 할 점

- 이 프로젝트는 API 연동형이 아니라 UI 자동화형입니다.
- 카카오톡 버전 변화에 따라 창 제목, 클래스명, 컨트롤 구조가 달라질 수 있습니다.
- 채팅방이 메인 창 내부 탭으로만 열려 있으면 탐지되지 않을 수 있습니다.
- 클립보드와 키 입력 시뮬레이션을 쓰므로 사용 중인 PC 상태에 영향을 받을 수 있습니다.

## 7. 수정 시 우선 확인할 포인트

- 채팅방 탐색이 실패하면 `ChatFinder.cs`의 창 필터 조건 확인
- 메시지 전송이 실패하면 `MessageSender.cs`의 입력 컨트롤 탐색과 키 입력 흐름 확인
- 카카오톡 업데이트 이후 동작이 깨지면 `Win32.GetClassName()` 기준으로 실제 자식 창 구조 재확인
- 문서와 코드가 어긋나지 않도록 변경 후 `README.md`와 이 문서도 같이 갱신
