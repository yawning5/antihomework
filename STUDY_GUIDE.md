# 학습 가이드

## 현재 구조를 이해하는 가장 빠른 순서

1. `MainForm.cs`
2. `SettingsStore.cs`
3. `PostgresClient.cs`
4. `MessageSender.cs`
5. `Win32.cs`

## 1. 왜 WinForms 클라이언트로 바꿨나

기존 CLI는 테스트에는 충분했지만, 실제 DB 연결 정보를 수시로 바꾸고 운영자가 직접 다루기에는 불편했습니다.

지금 구조는 다음 목적에 맞춰 바뀌었습니다.

- DB 접속 정보를 화면에서 수정
- 연결 테스트를 즉시 수행
- 조회 결과를 눈으로 확인
- 전송도 같은 클라이언트 안에서 수행

## 2. PostgreSQL 연결 구조

`PostgresClient.cs`는 Npgsql 드라이버를 사용합니다.

핵심 역할:

- 연결 문자열 생성
- 연결 테스트
- SQL 실행 후 `DataTable` 반환

지금은 사용자가 직접 SQL을 입력할 수 있게 되어 있으므로, 이후 운영 단계에서는 허용 쿼리 범위를 제한할지 검토가 필요합니다.

## 3. 설정 저장 구조

`SettingsStore.cs`는 `client-settings.json`을 읽고 씁니다.

장점:

- 실행 파일 폴더 기준으로 동작해서 배포가 단순함
- 운영자가 UI에서 설정을 저장하고 바로 재사용 가능

단점:

- 비밀번호가 평문 저장됨

## 4. 카카오톡 자동화 구조

자동화 코어는 기존과 비슷하게 유지됩니다.

- `ChatFinder.cs`
  - 카카오톡 메인 창 찾기
- `MessageSender.cs`
  - 검색 열기
  - 채팅방 선택
  - 메시지 붙여넣기
  - 전송

즉, UI는 바뀌었지만 카카오톡 자동화 모듈은 별도 계층처럼 남아 있습니다.

## 5. 실무적으로 먼저 보강할 부분

- DB 비밀번호 보관 방식
- 쿼리 제한 또는 템플릿화
- DB에서 읽은 데이터를 메시지 입력 영역에 매핑하는 로직
- 발송 성공/실패 로그 저장
- 카카오톡 포커스 안정화
